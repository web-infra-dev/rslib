# Overview

The `lib` field is an optional array of objects used to describe the library outputs generated by Rslib. Each `lib` item corresponds to one output and can contain lib configurations and [Rsbuild configurations](/config/rsbuild), as detailed in [Configure Rslib](/guide/basic/configure-rslib#configuration-structure).

- **Type:** `LibConfig[] | undefined` (see [`RslibConfig`](/api/javascript-api/types#rslibconfig))
- **Default:** `[{}]`
- **Required:** `false`

If `lib` is omitted, Rslib uses a default configuration equivalent to `lib: [{}]`; if `lib` is set explicitly, it must be a non-empty array.

For example, the following configuration will generate ESM and CJS outputs:

```js title="rslib.config.mjs"
export default {
  lib: [
    {
      format: 'esm',
    },
    {
      format: 'cjs',
    },
  ],
};
```

## Shared lib configurations

The following lib configurations can be placed outside the `lib` field as shared configuration for all outputs or inside a `lib` item to configure a specific output:

- [`autoExtension`](/config/lib/auto-extension)
- [`autoExternal`](/config/lib/auto-external)
- [`banner`](/config/lib/banner)
- [`bundle`](/config/lib/bundle)
- [`externalHelpers`](/config/lib/external-helpers)
- [`footer`](/config/lib/footer)
- [`outBase`](/config/lib/out-base)
- [`redirect`](/config/lib/redirect)
- [`shims`](/config/lib/shims)
- [`syntax`](/config/lib/syntax)

## Non-shared lib configurations

The following lib configurations do not support shared configuration and can only be specified in a `lib` item to configure an individual output:

- [`dts`](/config/lib/dts)
- [`experiments`](/config/lib/experiments)
- [`format`](/config/lib/format)
- [`id`](/config/lib/id)
- [`umdName`](/config/lib/umd-name)

# lib configuration

Lib configurations describe the library outputs generated by Rslib. `lib` is an array of objects, with each object representing one output:

```js title="rslib.config.mjs"
export default {
  lib: [{ format: 'esm' }, { format: 'cjs' }],
};
```

- **Type:** `LibConfig[]`
- **Default:** `undefined`
- **Required:** `true`

Each `lib` item supports both Rslib-specific lib configurations and [Rsbuild configurations](/config/rsbuild), allowing you to customize the build behavior of each output.

## Shared lib configurations

Some lib configurations can be specified at the top level of the Rslib configuration and shared by all `lib` items. To customize one output, override the shared configuration in the corresponding `lib` item:

```js title="rslib.config.mjs"
export default {
  // All outputs use ES2021 syntax and are bundleless by default
  syntax: 'es2021',
  bundle: false,
  lib: [
    { format: 'esm' },
    {
      format: 'cjs',
      // Only the CJS output is bundled
      bundle: true,
    },
  ],
};
```

The following lib configurations can be shared:

- [`bundle`](/config/lib/bundle)
- [`autoExtension`](/config/lib/auto-extension)
- [`autoExternal`](/config/lib/auto-external)
- [`redirect`](/config/lib/redirect)
- [`syntax`](/config/lib/syntax)
- [`externalHelpers`](/config/lib/external-helpers)
- [`banner`](/config/lib/banner)
- [`footer`](/config/lib/footer)
- [`shims`](/config/lib/shims)
- [`outBase`](/config/lib/out-base)

## Per-output lib configurations

The following lib configurations describe or customize an individual output and can only be specified in a `lib` item:

| Option                                   | Description                                     |
| ---------------------------------------- | ----------------------------------------------- |
| [`format`](/config/lib/format)           | Configure the module format of the output.      |
| [`id`](/config/lib/id)                   | Configure the unique identifier of the output.  |
| [`dts`](/config/lib/dts)                 | Configure TypeScript declaration generation.    |
| [`umdName`](/config/lib/umd-name)        | Configure the export name of the UMD output.    |
| [`experiments`](/config/lib/experiments) | Configure experimental features for the output. |

Shared lib configurations can also be specified in a `lib` item to customize that output.

## Rsbuild configurations

In addition to lib configurations, each `lib` item supports [Rsbuild configurations](/config/rsbuild). Top-level Rsbuild configurations apply to all outputs, while Rsbuild configurations inside a `lib` item customize that output.

See [Configure Rslib](/guide/basic/configure-rslib#configuration-structure) for details on using both types of configuration.

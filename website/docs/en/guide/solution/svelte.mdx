# Svelte

In this document, you will learn how to build a Svelte component library using Rslib.

## Create Svelte project

You can use `create-rslib` to create a project with Rslib + Svelte. Just execute the following command:

import { PackageManagerTabs } from '@theme';

<PackageManagerTabs
  command={{
    npm: 'npm create rslib@latest',
    yarn: 'yarn create rslib',
    pnpm: 'pnpm create rslib@latest',
    bun: 'bun create rslib@latest',
  }}
/>

Then select `Svelte` when prompted to "Select template", and select either `TypeScript` or `JavaScript` when prompted to "Select language".

::: note

1. The built-in `create-rslib` template provides Svelte 5 + TypeScript and Svelte 5 + JavaScript projects.

2. In the TypeScript template, Svelte component declarations are generated by the local `scripts/rslib-plugin-svelte-dts.ts` helper, which uses `svelte2tsx`. Therefore [lib.dts](/config/lib/dts) / [lib.redirect.dts](/config/lib/redirect#redirectdts) / [lib.banner.dts](/config/lib/banner#bannerdts) / [lib.footer.dts](/config/lib/footer#bannerdts) do not apply to the TypeScript template declaration output.

3. The TypeScript template publishes compiled JavaScript and declarations. The JavaScript template follows the React JavaScript template behavior and does not generate declaration files. Neither template publishes raw `.svelte` source files by default.

:::

## Use Rslib in an existing project

For developing Svelte components, you need to set the [target](/config/rsbuild/output#outputtarget) to `"web"` in `rslib.config.ts`. This is crucial because Rslib sets `target` to `"node"` by default, which is different from Rsbuild's default target value.

To compile Svelte (`.svelte` components), you need to register the [@rsbuild/plugin-svelte](https://rsbuild.rs/plugins/list/plugin-svelte) plugin. This plugin will automatically add the necessary configurations for Svelte build.

The recommended setup is to use bundle mode and let Rslib emit compiled ESM and extracted CSS:

```ts title="rslib.config.ts"
import { pluginSvelte } from '@rsbuild/plugin-svelte';
import { defineConfig } from '@rslib/core';
import { svelteDtsPlugin } from './scripts/rslib-plugin-svelte-dts';

export default defineConfig({
  resolve: {
    conditionNames: ['svelte', 'browser', '...'],
  },
  lib: [
    {
      format: 'esm',
      dts: false,
    },
  ],
  output: {
    target: 'web',
  },
  plugins: [
    pluginSvelte(),
    svelteDtsPlugin({
      declarationDir: './dist',
      libRoot: './src',
      tsconfig: 'tsconfig.json',
    }),
  ],
});
```

The generated package exports the compiled entry and an explicit CSS export:

```json title="package.json"
{
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.js"
    },
    "./style.css": "./dist/index.css"
  },
  "types": "./dist/index.d.ts"
}
```

Consumers can import the package and, when using the default extracted CSS output, import the stylesheet explicitly:

```ts
import { Button } from 'your-package';
import 'your-package/style.css';
```

## Declaration files

The TypeScript template disables Rslib's built-in declaration pipeline with `dts: false` and runs a local Rslib plugin after build:

```ts
svelteDtsPlugin({
  declarationDir: './dist',
  libRoot: './src',
  tsconfig: 'tsconfig.json',
});
```

This helper calls `svelte2tsx.emitDts` and emits bundleless-style declaration files, for example:

```txt
dist/index.d.ts
dist/Button.svelte.d.ts
```

Because declarations are generated by `svelte2tsx`, Rslib declaration options such as `dts.bundle`, `redirect.dts`, `banner.dts`, and `footer.dts` are not part of the TypeScript template's declaration flow. The JavaScript template does not generate declaration files.

## Bundleless mode

Bundleless mode is not the default recommendation for Svelte libraries. The default Svelte loader CSS extraction flow can produce invalid loader-request imports in bundleless output, and an index-only bundleless entry can reference a missing component chunk.

If you still want bundleless JavaScript output, disable CSS extraction in the Svelte loader options:

```ts title="rslib.config.ts"
export default defineConfig({
  lib: [
    {
      bundle: false,
      format: 'esm',
      dts: false,
    },
  ],
  output: {
    target: 'web',
  },
  plugins: [
    pluginSvelte({
      svelteLoaderOptions: {
        emitCss: false,
      },
    }),
    svelteDtsPlugin(),
  ],
});
```

With `emitCss: false`, Svelte component styles are injected by the compiled component JavaScript instead of being emitted as `dist/index.css`, so you should remove the `./style.css` export from `package.json`.

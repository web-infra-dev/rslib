# Solid

在本文档中，你将学习如何使用 Rslib 构建 Solid 组件库，你可在 [示例](https://github.com/rstackjs/rstack-examples/tree/main/rslib) 中查看 Solid 相关演示项目。

## 创建 Solid 项目

你可以使用 `create-rslib` 创建 Rslib + Solid 项目。只需执行以下命令：

import { PackageManagerTabs } from '@theme';

<PackageManagerTabs
  command={{
    npm: 'npm create rslib@latest',
    yarn: 'yarn create rslib',
    pnpm: 'pnpm create rslib@latest',
    bun: 'bun create rslib@latest',
  }}
/>

然后，当提示 "Select template" 时选择 `Solid`。

## 版本状态

内置 Solid 模板面向 Solid v1，这是大多数生产 Solid 组件库使用的稳定版本。

| Solid 版本 | Rsbuild 插件 | 状态 |
| ---------- | ------------ | ---- |
| `solid-js@^1` | `@rsbuild/plugin-solid@^1.2.2` | 稳定版本，内置 Rslib Solid 模板默认使用。 |
| `solid-js@2.0.0-beta` + `@solidjs/web@2.0.0-beta` | `@rsbuild/plugin-solid@2.0.0-beta.0` | 实验版本。Solid 2 和 Rsbuild Solid v2 插件当前都处于 beta 阶段。 |

Solid v1 和 v2 的产物形态一致：为普通消费端编译一份 ESM 输出，同时通过 `"solid"` 导出条件发布一份保留 JSX 的源码输出。Solid v2 主要区别在于依赖范围和运行时包。

## 在现有 Rslib 项目中使用

开发 Solid 组件库，需要在 `rslib.config.ts` 中设置 [target](/config/rsbuild/output#outputtarget) 为 `"web"`。这一点至关重要，因为 Rslib 默认将 `target` 设置为 `"node"`。

要编译 Solid JSX，你需要同时注册 [@rsbuild/plugin-babel](https://rsbuild.rs/zh/plugins/list/plugin-babel) 和 [@rsbuild/plugin-solid](https://rsbuild.rs/zh/plugins/list/plugin-solid) 插件。Solid 插件使用 `babel-preset-solid` 将 JSX 转换为 Solid 的运行时 DOM 表达式。

```json title="package.json"
{
  "devDependencies": {
    "@rsbuild/plugin-solid": "^1.2.2",
    "solid-js": "^1.9.9"
  },
  "peerDependencies": {
    "solid-js": "^1.8.0"
  }
}
```

```ts title="rslib.config.ts"
import { pluginBabel } from '@rsbuild/plugin-babel';
import { pluginSolid } from '@rsbuild/plugin-solid';
import { defineConfig } from '@rslib/core';

export default defineConfig({
  source: {
    entry: {
      index: ['./src/**'],
    },
  },
  lib: [
    {
      bundle: false,
      dts: true,
      format: 'esm',
      plugins: [
        pluginBabel({
          include: /\.(?:jsx|tsx)$/,
        }),
        pluginSolid(),
      ],
    },
  ],
  output: {
    target: 'web',
  },
});
```

## 双输出与 "solid" 导出条件

Solid 组件库通常同时提供**编译后的输出**和**保留 JSX 的源码输出**。`package.json` 中的 `"solid"` 导出条件指向未编译的 JSX 源码，允许下游打包器（如配置了 `@rsbuild/plugin-solid` 的项目）以最优的方式本地编译（DOM vs SSR）。

添加第二个 lib 配置来保留 JSX。第一个 lib 通过 `babel-preset-solid` 编译 JSX，第二个则通过 SWC 的 `runtime: 'preserve'` 保持原始 JSX 不变，并启用 Rspack 的 JSX parser：

```ts title="rslib.config.ts"
import { pluginBabel } from '@rsbuild/plugin-babel';
import { pluginSolid } from '@rsbuild/plugin-solid';
import { defineConfig } from '@rslib/core';

export default defineConfig({
  source: {
    entry: {
      index: ['./src/**'],
    },
  },
  lib: [
    {
      bundle: false,
      dts: true,
      format: 'esm',
      plugins: [
        pluginBabel({
          include: /\.(?:jsx|tsx)$/,
        }),
        pluginSolid(),
      ],
    },
    {
      bundle: false,
      format: 'esm',
      output: {
        distPath: {
          root: './dist/source',
        },
        filename: {
          js: '[name].jsx',
        },
      },
      tools: {
        swc: {
          detectSyntax: 'auto',
          jsc: {
            transform: {
              react: {
                runtime: 'preserve',
              },
            },
          },
        },
        rspack: {
          module: {
            parser: {
              javascript: {
                jsx: true,
              },
            },
          },
        },
      },
    },
  ],
  output: {
    target: 'web',
  },
});
```

- 第一个 lib 使用 `pluginBabel` + `pluginSolid` 将 JSX 编译为 Solid 的运行时调用（如 `template()`、`insert()`、`delegateEvents()`），产出标准 `.js` 文件。
- 第二个 lib 跳过 Solid 编译。`jsc.transform.react.runtime: 'preserve'` 告诉 SWC 保持 JSX 语法不变，`module.parser.javascript.jsx: true` 告诉 Rspack parser 接受输出中的 JSX。

然后在 `package.json` 中配置 exports：

```json title="package.json"
{
  "exports": {
    ".": {
      "solid": "./dist/source/index.jsx",
      "types": "./dist/index.d.ts",
      "import": "./dist/index.js"
    }
  }
}
```

当下游应用使用 `@rsbuild/plugin-solid` 时，`"solid"` 条件会自动加入模块解析，打包器会解析到 `.jsx` 源码并本地编译。不使用 Solid 感知打包器的应用会回退到预编译的 `"import"` 入口。

## TypeScript

对于使用 TypeScript 的 Solid v1 项目，在 `tsconfig.json` 中设置 `"jsx": "preserve"` 和 `"jsxImportSource": "solid-js"`：

```json title="tsconfig.json"
{
  "compilerOptions": {
    "jsx": "preserve",
    "jsxImportSource": "solid-js"
  }
}
```

对于 Solid v2 beta 项目，JSX runtime 位于 `@solidjs/web`，因此需要将 `jsxImportSource` 设置为 `@solidjs/web`：

```json title="tsconfig.json"
{
  "compilerOptions": {
    "jsx": "preserve",
    "jsxImportSource": "@solidjs/web"
  }
}
```

## 使用 Solid v2 beta

Solid v2 和 `@rsbuild/plugin-solid@2.0.0-beta.0` 当前都处于 beta 阶段。默认 Rslib Solid 模板不面向 v2，但如果你明确要测试 beta 运行时，可以手动调整模板配置。

修改 `package.json` 里的依赖范围：

```json title="package.json"
{
  "devDependencies": {
    "@rsbuild/plugin-solid": "2.0.0-beta.0",
    "@solidjs/web": ">=2.0.0-beta.0 <2.0.0-experimental.0",
    "solid-js": ">=2.0.0-beta.0 <2.0.0-experimental.0"
  },
  "peerDependencies": {
    "@solidjs/web": ">=2.0.0-beta.0 <2.0.0-experimental.0",
    "solid-js": ">=2.0.0-beta.0 <2.0.0-experimental.0"
  }
}
```

TypeScript 项目还需要修改 `tsconfig.json`：

```json title="tsconfig.json"
{
  "compilerOptions": {
    "jsx": "preserve",
    "jsxImportSource": "@solidjs/web"
  }
}
```

如果你的库或示例直接渲染到 DOM，从 `@solidjs/web` 导入 DOM runtime API，而不是 `solid-js/web`：

```ts
import { render } from '@solidjs/web';
```

Rslib 构建配置可以保持不变。`@rsbuild/plugin-solid@2.0.0-beta.0` 仍然使用 Babel 和 `babel-preset-solid`，v2 的区别主要是编译器和运行时依赖集合。

## SSR

大多数 Solid 组件库默认不会单独发布 SSR 构建。下游框架通常会解析 `"solid"` 条件，拿到保留 JSX 的源码输出，再按自己的目标编译。

如果你确实需要预编译的 SSR 产物，可以在额外的 lib 配置中通过 `pluginSolid()` 传入 `solid` 选项：

```ts
pluginSolid({
  solid: {
    generate: 'ssr',
    hydratable: true,
  },
})
```

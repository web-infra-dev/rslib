# WebAssembly

Rslib 支持遵循 [WebAssembly ESM Integration](https://github.com/WebAssembly/esm-integration) 提案语法使用的 WebAssembly（`.wasm`）模块。

:::warning

使用 WebAssembly ESM Integration 提案语法的源码仅能构建为 ESM 产物，因此 [format](/config/lib/format) 须为 `'esm'`（默认值）。

:::

## 导入 WASM 模块

Rslib 支持以下几种 `.wasm` 模块的导入形式。

### 静态导入与导出

通过标准的 ESM 语法导入 `.wasm` 模块并访问实例化后的导出：

```ts
import { add } from './add.wasm';
import * as wasm from './add.wasm';
import './add.wasm';

export { add } from './add.wasm';
export * from './add.wasm';
export * as wasm from './add.wasm';
```

### 动态导入

```ts
const wasm = await import('./add.wasm');
```

### Source phase 导入

[`import source`](https://github.com/tc39/proposal-source-phase-imports) 是 WebAssembly ESM Integration 的一部分，它导入的是编译后的 `WebAssembly.Module`，可以让你使用自定义 import object 手动完成实例化：

```ts
import source addModule from './add.wasm';

const { instance } = await WebAssembly.instantiate(addModule, {
  env: { now: Date.now },
});
```

也支持通过 `import.source()` 进行动态 source phase 导入：

```ts
const addModule = await import.source('./add.wasm');
```

:::note

TypeScript 目前不解析 `import source` 与 `import.source()`。请将 source phase 导入写在 JavaScript 文件中，或使用支持该语法的工具链。

:::

## 产物

你可以通过 [lib.wasm.mode](/config/lib/wasm#wasmmode) 控制 `.wasm` 模块的产物形态：

- **Compile**：Rslib 解析 `.wasm` 模块，生成加载和实例化 WebAssembly 的 JavaScript 胶水代码，并将二进制文件作为静态资源输出。消费方无需支持 ESM Integration。
- **Preserve**：Rslib 在产物 JavaScript 中保留 `.wasm` import，并将二进制文件原样输出。下游构建工具或运行时负责解析和加载 `.wasm` 模块。Preserve 模式要求 [bundle](/config/lib/bundle) 为 `false`。

| bundle  | 默认 mode  | 可选 mode              |
| ------- | ---------- | ---------------------- |
| `true`  | `compile`  | 仅 `compile`           |
| `false` | `preserve` | `compile` 或 `preserve` |

如果产物被浏览器等运行时直接消费，使用 Compile 模式。如果 bundleless 产物会交给支持 ESM Integration 的构建工具（如 Rsbuild）继续处理，或目标运行时（如 [Node.js](https://nodejs.org/api/esm.html#wasm-modules) `^22.19.0 || >=24.5.0`）原生支持该特性，可以使用 Preserve 模式。

### Compile mode

二进制文件输出到 [output.distPath.wasm](/config/rsbuild/output#outputdistpath)（默认 `static/wasm`），文件名带 content hash：

<Tabs>
<Tab label="bundle">

```
dist
├── index.js
└── static
    └── wasm
        └── [contenthash].module.wasm
```

</Tab>
<Tab label="bundleless">

```
dist
├── index.js
├── utils.js
└── static
    └── wasm
        └── [contenthash].module.wasm
```

</Tab>
</Tabs>

生成的加载运行时由 [output.target](/config/rsbuild/output#outputtarget) 决定：

- `web` target 会通过 `fetch` 加载 `.wasm`。
- `node` target 会通过 Node.js 的异步文件系统 API 加载 `.wasm`。

### Preserve mode

`.wasm` 文件按源码相对路径和原文件名输出：

```
dist
├── index.js
└── add.wasm
```

```js title="dist/index.js"
import { add } from './add.wasm';

export { add };
```

文件名不受 [output.filenameHash](/config/rsbuild/output#outputfilenamehash) 影响。

Rslib 会更新 JavaScript 中的 import 使其指向输出后的 `.wasm` 文件，包括通过 [output.filename.js](/config/rsbuild/output#outputfilename) 将 JavaScript 输出到其他目录的情况。但 WebAssembly 二进制内部的 import 保持原样——如果 `.wasm` 反向导入 JavaScript（如 wasm-bindgen `bundler` target），请避免以下会改变 JavaScript 相对路径或文件名的配置：

- [output.distPath.js](/config/rsbuild/output#outputdistpath)（将 JS 移到子目录）
- [output.filename.js](/config/rsbuild/output#outputfilename)（改变文件名模板）
- [autoExtension](/config/lib/auto-extension)（将 `.js` 改为 `.mjs` 等）

## 使用 wasm-bindgen

[`wasm-bindgen`](https://wasm-bindgen.github.io/wasm-bindgen/) 是使用 Rust 构建 WebAssembly 库的工具，可通过 `--target` 选项生成面向不同运行环境的产物。

Rslib 支持以下 target：

- [`bundler`](#bundler)（推荐）
- [`module`](#module)
- [`web` / `experimental-nodejs-module`](#web--experimental-nodejs-module)

### `bundler`

胶水 JavaScript 文件以 ES module 形式导入 `.wasm` 文件，`.wasm` 模块又反向导入胶水 JavaScript 文件以获得 imports 对象。这种双向依赖意味着 Preserve 模式下存在[反向引用约束](#preserve-mode)——需要保持胶水 JavaScript 文件的相对路径和文件名不变。Compile 模式则不受影响。

### `module`

胶水 JavaScript 文件通过 [source phase 导入](#source-phase-导入)语法导入 `.wasm` 文件，并完全在 JavaScript 中构造 imports 对象。由于 `.wasm` 不会反向导入 JavaScript，不存在反向引用约束，无需关注文件名或路径配置，Compile 和 Preserve 模式都能直接工作。

### `web` / `experimental-nodejs-module`

胶水 JavaScript 文件通过 `new URL('./pkg.wasm', import.meta.url)` 加载 `.wasm` 文件。Rslib 将其作为静态资源处理，而不是 WebAssembly ESM 模块。

此时 `wasm.mode` 不适用。

## 类型声明

TypeScript 未内置 `.wasm` 文件的模块声明。使用 TypeScript 时，你需要在 `.wasm` 文件旁添加一个 `.d.wasm.ts` 声明文件，并在 `tsconfig.json` 中启用 [`allowArbitraryExtensions`](https://www.typescriptlang.org/tsconfig/allowArbitraryExtensions.html)：

```ts title="src/add.d.wasm.ts"
export function add(a: number, b: number): number;
```

# Wasm

Rslib 原生支持 WebAssembly（WASM）模块，允许你在项目中通过 [WebAssembly ESM Integration](https://github.com/WebAssembly/esm-integration) 提案定义的 ESM 导入方式，直接导入和使用 `.wasm` 文件。

:::warning

使用 WebAssembly ESM Integration 导入 `.wasm` 模块时，仅支持生成 ESM 格式的产物，因此 [format](/config/lib/format) 必须设置为 `'esm'`（默认值）。

:::

## 导入 WASM 模块

Rslib 支持以下几种 `.wasm` 模块的导入形式。

### 静态导入与导出

通过标准的 ESM 语法导入 `.wasm` 模块并访问实例化后的导出：

```ts
import { add } from './add.wasm';
import * as wasm from './add.wasm';
import './add.wasm';

export { add } from './add.wasm';
export * from './add.wasm';
export * as wasm from './add.wasm';
```

### 动态导入

```ts
const wasm = await import('./add.wasm');
```

### Source phase 导入

你可以使用 [Source Phase Imports](https://github.com/tc39/proposal-source-phase-imports) 获取编译后的 `WebAssembly.Module`，并通过自定义 import object 手动完成实例化：

```ts
import source addModule from './add.wasm';

const { instance } = await WebAssembly.instantiate(addModule, {
  env: { now: Date.now },
});
```

你也可以使用 `import.source()` 动态获取编译后的 `WebAssembly.Module`：

```ts
const addModule = await import.source('./add.wasm');
```

:::note

TypeScript 目前无法解析 `import source` 和 `import.source()`。请在 JavaScript 文件中使用这些语法，或选择支持它们的工具链。

:::

## 输出模式

你可以通过 [lib.wasm.mode](/config/lib/wasm#wasmmode) 选择 `.wasm` 模块的输出模式。具体使用哪种模式，取决于 [bundle](/config/lib/bundle) 配置以及消费方是否支持 WebAssembly ESM Integration：

- 当 `bundle` 为 `true` 时，仅支持 `compile` 模式。
- 当 `bundle` 为 `false` 时：
  - 如果产物由支持 WebAssembly ESM Integration 的下游构建工具（如 Rsbuild），或原生支持该特性的目标运行时（如 [Node.js](https://nodejs.org/api/esm.html#wasm-modules) `^22.19.0 || >=24.5.0`）消费，使用 `preserve` 模式（默认）。消费方负责解析和加载 `.wasm` 模块。
  - 如果消费方不支持 WebAssembly ESM Integration，或不希望依赖消费方处理 `.wasm` 模块，使用 `compile` 模式。

### Compile 模式

Rslib 解析每个 `.wasm` 模块，生成加载和实例化该模块所需的 JavaScript 胶水代码，并将二进制文件作为静态资源输出。

二进制文件输出到 [output.distPath.wasm](/config/rsbuild/output#outputdistpath)（默认 `static/wasm`），文件名带 content hash：

<Tabs>
<Tab label="bundle">

```
dist
├── index.js
└── static
    └── wasm
        └── [contenthash].module.wasm
```

</Tab>
<Tab label="bundleless">

```
dist
├── index.js
├── utils.js
└── static
    └── wasm
        └── [contenthash].module.wasm
```

</Tab>
</Tabs>

生成的加载代码取决于 [output.target](/config/rsbuild/output#outputtarget)：

- `web`：使用 `fetch` 加载 `.wasm` 文件。
- `node`：使用 Node.js 异步文件系统 API 加载 `.wasm` 文件。

### Preserve 模式

Rslib 在 JavaScript 产物中保留对 `.wasm` 模块的 import 语句，并原样输出二进制文件。

`.wasm` 文件按源码相对路径和原文件名输出：

```
dist
├── index.js
├── utils.js
└── add.wasm
```

```js title="dist/index.js"
import { add } from './add.wasm';

export { add };
```

:::note .wasm 导入 JavaScript 时的路径限制

在 `preserve` 模式下，`.wasm` 文件名不受 [output.filenameHash](/config/rsbuild/output#outputfilenamehash) 影响。

Rslib 会更新 JavaScript 中对 `.wasm` 文件的 import，使其指向输出后的文件，但不会修改 WebAssembly 二进制内部的 import。如果 `.wasm` 模块反向导入 JavaScript（例如使用 wasm-bindgen 的 `bundler` target），请避免使用以下会改变 JavaScript 输出路径或文件名的配置：

- [output.distPath.js](/config/rsbuild/output#outputdistpath)
- [output.filename.js](/config/rsbuild/output#outputfilename)
- [autoExtension](/config/lib/auto-extension)

:::

## 使用 wasm-bindgen

[`wasm-bindgen`](https://wasm-bindgen.github.io/wasm-bindgen/) 是使用 Rust 构建 WebAssembly 库的工具，可通过 `--target` 选项生成面向不同运行环境的产物。

Rslib 支持以下 target：

- [`bundler`](#bundler)（推荐）
- [`module`](#module)
- [`web` / `experimental-nodejs-module`](#web--experimental-nodejs-module)

### `bundler`

胶水 JavaScript 文件以 ES module 形式导入 `.wasm` 文件，`.wasm` 模块又反向导入胶水 JavaScript 文件以获得 imports 对象。这种双向依赖意味着 Preserve 模式需要遵守[上述路径限制](#preserve-模式)，即保持胶水 JavaScript 文件的相对路径和文件名不变。Compile 模式不受此限制。

### `module`

胶水 JavaScript 文件通过 [source phase 导入](#source-phase-导入)语法导入 `.wasm` 文件，并完全在 JavaScript 中构造 imports 对象。由于 `.wasm` 不会导入 JavaScript，因此无需保持 JavaScript 文件的输出路径和文件名，Compile 和 Preserve 模式都能直接工作。

### `web` / `experimental-nodejs-module`

胶水 JavaScript 文件通过 `new URL('./pkg.wasm', import.meta.url)` 加载 `.wasm` 文件。Rslib 将其作为静态资源处理，而不是 WebAssembly ESM 模块。

此时 `wasm.mode` 不适用。

## 类型声明

TypeScript 未内置 `.wasm` 文件的模块声明。使用 TypeScript 时，你需要在 `.wasm` 文件旁添加一个 `.d.wasm.ts` 声明文件，并在 `tsconfig.json` 中启用 [`allowArbitraryExtensions`](https://www.typescriptlang.org/tsconfig/allowArbitraryExtensions.html)：

```ts title="src/add.d.wasm.ts"
export function add(a: number, b: number): number;
```
